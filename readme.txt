Даний бот використовувався для ефективного і швидкого обміну скінів CS:GO між двома платформами - CS.MONEY і CSGO.EMPIRE
Всі дії велися в потоках з двох акаунтів через потребу паралельних ставок, оскільки CSGO.EMPIRE - рулетка.

off.py - скріпт, що приймав всі запити на обмін для стім акаунта юзера (запускалось для кожного акаунта окремо).

withdraw_db.py - скріпт, що постійно викачував json дані з сайтів, щоб завжди мати актуальні дані (і не втрачати швидкість виконання інших скріптів, які би потребували доступ до даних), в даній версії бота всі дані зберігались в pickle файл.

maFile - файли з даними, потрібними для доступу до SteamAPI.

cookies.json - більшість скріптів брали актуальні кукі звідси, оновлювались рідко, але так, як вся система потребувала кукі, досить зручно було один раз їх оновлювати для всіх скріптів.

Папка bot + hustle.py - для виводу в телеграм.

knapsack.py(x) і всі файли формату .c, .pyd, і папка build - перекомпільований за допомогою Cython під С алгоритм наповнення рюкзака (тут вагою бралась ціна скіна на CSGO.EMPIRE, оскільки цей алгоритм використовується лише в separator.py, що міняв одну дорогу річ на CS.MONEY на багато дешевих, але вигідних для обміну).

empauto.py - автоматичний залив всіх речей на депозит на CSGO.EMPIRE

keys_exchanger.py - скріпт, що контролював банк, що весь час обмінюєтся шляхом обміну зайвих речей на ключі, які потім виводились за допомогою ботів на Приват.

divider.py - поділ речей по ціні навпіл перед початком роботи separator.py і перед empauto.py.

withdraw.py - автоматичний вивід з депозиту речами в інвентар СТІМ.

siziph.py - власне сама програма, що запускала все, окрім off.py i withdraw_db.py в потоках для роботи для двох акаунтів по черзі, яка була фізично прописана в tasks.json, оскільки на ранніх етапах у нас виникла проблема з синхронізуванням потоків.

tradelist - тут були записані речі, які на даний момент в інвентарі стім і кожній присвоювались значення: to_money (обмін на ключі), to_emp (для депозиту на CSGO.EMPIRE), for_parse (для обміну на CS.MONEY).

userx.pickle - тут зберігались об'єкти сесії стім АРІ, тому що стім карає абуз їх ресурсів і системи.

Папка logs - ієрархія логів залежно від виконуваного скріпта (кожен скріпт писав детальні логи).

Папка nodejs - програма для синхронізованих автоматичних ставок з двох акаунтів через сокет CSGO.EMPIRE.

Всі дії даною системою є законними, а сама система слугує лише як приклад властового мені (хоча це не найкращий приклад, бо писався поспіхом і без систем контролю версій) стилю написання коду і показником стеку технологій, якими я в тій чи іншій мірі володію (nodejs сокет - не моя частина проекту).

Відразу хочу перепросити за відсутність коментарів, деякі там навіть були, але з часом повитерались через багаторазове переписування.
